Council Config

Set your project's theme, preset tiers, and per-role standing rules. These are written to .council/ and used in every council session for this project.

Theme

Pick the visual theme for council output and this config window.

Presets

Select the tiers that match your project's exposure and audience. These set floors for every role in this session.

Security

Select the tier that matches your project's exposure.

Floor: no secrets in git, real auth, row-level access. This applies regardless of tier.
Accessibility

Select the WCAG conformance level that fits your audience.

Floor: AA on public UI. This applies regardless of tier.
Compliance

Select the compliance posture that matches your data handling.

Floor: no PII in logs/client, lawful basis to hold data. This applies regardless of tier.

House Rules

Any standing rules for a role? For example: always use [technology], never propose [pattern], assume [constraint]. Leave blank to use the default rubric. Click a role to expand it.
